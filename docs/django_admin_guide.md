# Руководство по Django Admin

URL: `https://<VPS_IP>/admin/`  
В разработке: `http://localhost:8000/admin/`

---

## Вход

Логин и пароль создаются командой:
```bash
# Dev:
docker-compose -f docker-compose.yml -f docker-compose.dev.yml exec web python manage.py createsuperuser

# Prod:
docker-compose exec web python manage.py createsuperuser
```

---

## Разделы

### Пользователи (`/admin/users/user/`)

Основная таблица всех пользователей бота.

**Поля в списке:**
- `telegram_id` — уникальный идентификатор в Telegram
- `display_name` — имя (первое + фамилия из Telegram)
- `telegram_username` — @username (может отсутствовать)
- `role` — `admin` / `worker`
- `status` — `pending` / `active` / `inactive` / `banned`
- `is_activated` — был ли активирован через invite key
- `balance` — баланс в рублях
- `attracted_count` — количество привлечённых людей (устанавливается вручную)
- `work_url` — рабочая ссылка воркера

**Фильтры:** по роли, статусу, is_activated, is_blocked_bot.

**Поиск:** по telegram_id, first_name, last_name, telegram_username.

**Редактирование пользователя:**
- Можно изменить: `role`, `status`, `is_activated`, `balance`, `work_url`, `attracted_count`, `referred_by`
- Не рекомендуется менять `telegram_id` — это первичный ключ для идентификации пользователя в Telegram

> Изменение статуса через Django Admin не уведомляет пользователя в боте. Для уведомления используйте бот.

---

### Invite Keys (`/admin/invites/invitekey/`)

Управление ключами приглашения.

**Поля:**
- `key` — сам ключ (генерируется автоматически, UUID-стиль)
- `label` — метка для удобства (не видна пользователям)
- `is_active` — включён/выключен
- `max_uses` — лимит использований (пусто = безлимит)
- `uses_count` — сколько раз уже использован
- `expires_at` — дата истечения (пусто = бессрочный)

**Создание ключа:**
1. Нажмите «Добавить Invite Key»
2. `label` — внутреннее название (например: «Набор апрель 2026»)
3. `max_uses` — оставьте пустым для безлимита
4. `expires_at` — оставьте пустым для бессрочного
5. Сохраните → скопируйте сгенерированный `key`

**Инлайн: Активации (InviteActivation)** — таблица внутри карточки ключа показывает, кто и когда активировал этот ключ.

---

### Активации ключей (`/admin/invites/inviteactivation/`)

Лог всех активаций. Только чтение — не создавайте записи вручную.

**Поля:** ключ, пользователь, дата активации.

---

### Рефералы (`/admin/referrals/referrallink/`)

Реферальные ссылки пользователей. Создаются автоматически при первом обращении пользователя к своей ссылке.

**Поля:** пользователь, токен (уникальный UUID), дата создания.

Не удаляйте записи — это сломает существующие реферальные ссылки.

---

### Настройки рефералов (`/admin/referrals/referralsettings/`)

Единственная запись (singleton). Содержит:
- `rate_percent` — процент с заработка реферала (Decimal, 0–100)
- `updated_by` — кто последний изменил

Изменить также можно через бот: `/admin` → «⚙️ Настройки» → «✏️ Изменить ставку».

---

### Статистика (`/admin/stats/userdailystats/`)

Дневная статистика по задачам каждого воркера.

**Поля:**
- `user` — воркер
- `date` — дата
- `tasks_submitted` — заявок отправлено
- `tasks_completed` — выполнено
- `completion_rate` — % выполнения (вычисляемое свойство)

Записи создаются автоматически при выполнении задач. Можно редактировать вручную для корректировок.

---

### Рассылки (`/admin/broadcasts/broadcast/`)

**Статусы рассылки:**
- `draft` — черновик (создана, но не подтверждена)
- `confirmed` — подтверждена, готова к запуску
- `running` — выполняется (Celery task активен)
- `done` — завершена успешно
- `failed` — завершена с ошибкой

**Аудитория:**
- `all` — все пользователи
- `active` — только активные
- `invited` — прошедшие активацию через invite key

> Запускать рассылку через Django Admin не рекомендуется — используйте бот (раздел «📢 Рассылки»), там есть подтверждение и предпросмотр количества получателей.

---

### Логи доставки (`/admin/broadcasts/broadcastdeliverylog/`)

Детальный лог каждой отправки.

**Статусы:** `sent` / `failed` / `blocked`

Фильтр по рассылке и статусу позволяет быстро найти проблемные доставки.

---

### Периодические задачи (`/admin/django_celery_beat/`)

Управление расписанием Celery Beat. Здесь можно:
- Создавать периодические задачи (например, ежедневный отчёт)
- Видеть историю выполнения

Стандартный раздел django-celery-beat, изменяйте с осторожностью.

## Частые операции

### Создать администратора из Django Admin

1. Найдите пользователя в `Users`
2. Измените `role` → `admin`
3. Установите `is_activated` = True, `status` = `active`
4. Сохраните

> Обязательно сначала пусть пользователь написал боту `/start` — иначе его нет в базе.

### Сбросить баланс

1. Откройте карточку пользователя
2. Измените поле `balance`
3. Сохраните

### Деактивировать воркера

1. Найдите пользователя
2. `status` → `inactive` или `banned`
3. Сохраните

Пользователь не получит уведомление автоматически — при следующем обращении к боту увидит соответствующий статус.

### Посмотреть кто использовал ключ

Откройте `Invite Keys` → выберите ключ → внизу карточки раздел «Активации».

## Заметки

- Тема оформления: **django-unfold** (тёмный режим, современный интерфейс)
- Язык: русский (`LANGUAGE_CODE = "ru"`)
- Статика обслуживается через **whitenoise** (не нужен отдельный веб-сервер для `/static/`)
